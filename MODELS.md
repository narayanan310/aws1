# AI Models

AI Photo Manager relies on locally executed Machine Learning models to analyze images and generate search embeddings, guaranteeing that your personal photos never leave your machine.

## Vision-Language Model (VLM)

The VLM is responsible for looking at the image and generating all the metadata (caption, tags, objects, OCR).

### Default Model: Qwen2.5-VL-3B-Instruct
- **Architecture**: A multimodal autoregressive model by Alibaba Cloud.
- **Why it was chosen**: Qwen2.5-VL natively understands arbitrary image aspect ratios and resolutions, making it vastly superior to older models that crop or squish images to square grids (like 336x336). It also has state-of-the-art OCR capabilities.
- **Size**: ~6 GB on disk.
- **VRAM Required**: ~8 GB at 16-bit precision.
- **Performance**: 
  - NVIDIA GPU (RTX 3060+): ~1-3 seconds per image.
  - Apple Silicon (M1/M2/M3): ~2-5 seconds per image.
  - Pure CPU: ~30-90 seconds per image.

### Alternatives Considered
- **LLaVA 1.5 (7B/13B)**: Too large for many consumer GPUs, and struggles with non-square aspect ratios compared to Qwen.
- **Moondream2**: Much smaller (under 2B parameters) and faster, but generates significantly lower-quality captions and struggles with OCR.

### How it is loaded
The model is loaded dynamically by the background worker using HuggingFace `transformers`. It uses `device_map="auto"`, which automatically delegates computation to your GPU/MPS if available, and falls back to CPU if not.

## Embedding Model

The Embedding model converts the text tags and descriptions generated by the VLM into 384-dimensional mathematical vectors. This allows for Semantic Search (e.g., searching for "water" will return photos tagged with "ocean" or "lake").

### Default Model: all-MiniLM-L6-v2
- **Architecture**: A compressed BERT model trained for semantic similarity.
- **Why it was chosen**: It is extremely fast, very small, and provides excellent baseline performance for English text semantic matching.
- **Size**: ~90 MB on disk.
- **VRAM Required**: < 500 MB.

## Customizing Models

Currently, the models are hardcoded in the `infrastructure/ml` layer. 
However, the UI Settings dashboard includes a dropdown for "Preferred Vision Model". Future updates will dynamically parse this setting to allow hot-swapping between models like LLaVA and Qwen.

To upgrade or replace the model manually, you can edit `app/infrastructure/ml/vision_model.py` and replace `Qwen/Qwen2.5-VL-3B-Instruct` with the HuggingFace ID of your preferred compatible model.
